Functional Specifications

Form Error Handling

This functional specification establishes the need for a human-computer interaction (HCI) dataset and algorithm to translate visual error syntax into screen-reader friendly labels which can provide people who are blind with the critical information needed to successfully complete forms on the web. In addition to justifying the dataset creation, this document proposes a method and guidelines for creating said dataset. Link to specification.

Pop-Up Focus

This functional specification outlines a possible method for making pop-ups more accessible for screen reader users who are blind. The dataset that this proposes would help train a machine-learning algorithm to identify: When a pop-up has appeared, the type of pop-up, and the exit mechanism for the pop-up. Link to specification.

Transcending Linear Navigation

We have identified four key actions that are instrumental in navigation: 1) browsing through search results 2) accessing filters 3) finding the “my cart” button and 4) finding the search bar. This functional specification proposes an online shopping experience designed specifically for blind users that capitalizes on this structure, which will allow users to switch between shopping tasks dynamically, in a predictable way. Link to specification.
